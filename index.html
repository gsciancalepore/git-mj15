Areas de GIT

Working Directory

El área de trabajo,basicamente consiste en la carpeta donde inicializamos el repositorio y donde vamos a estar creando, editando y eliminando los archivos de nuestro proyecto. Muy rar vez vamos a estar utilizando la linea de comandos en esta área.

  • dir || ls muestra el contenido de la carpeta
  • mkdir <name> crea una carpeta
  • cd <path> abre un directorio
  • git init inicializa un repositorio de git
  • rm -rf .git elimina el repositorio

Staging Area (INDEX)

Area de control de cambios es quella donde se realiza el seguimiento de los archivos y permite la comparación del Working Directory con los commits almacenados en el historial de cmambios. Cuando agregamos un archivo al INDEX se realiza la captura del código (SNAPSHOT) y luego debemos confirmar si vamos a utilizarla o agregar mas modificaciones.

  • git status muestra al usuario el estado de los diferentes archivos que se encuentran en la carpeta del proyecto, dentro de los valores que podemos encontrar están:
    • untracked: no se está haciendo seguimiento, irrecuperable en caso de pérdida
    • staged: se realizó la captura, pero todavía no se confirmaron los cambios
    • commit: los cambios fueron registrados en el historial mediante un commit
    • modified: el archivo difiere con la última captura realizada, se está haciendo seguimiento
  • git add <file> agrega el archivo al INDEX
  • git diff muestra las diferencias entre el Working Directory y la última confirmación
  • git restore --stage <file> quita la captura realizada del INDEX
  • git rm --catched <file> elimina el archivo del INDEX, no realiza mas seguimiento.

Repository

Corresponde al area donde se almacenan las confirmaciones de cambios en formato BLOB (Binary Large Object File), una vez realizadas creamos un punto de control al que podemos volver en cualquier momento. Estas commits se utilizan como referentes para navegar dentro del historial.

  • git commit abre el editor para realizar una confirmacion
  • git log muestra el historial de confirmaciones
  • git blame <file> muestra los datos del autor que edito el archivo linea por linea
  • git push -u <remote> <branch> agrega al repositorio remoto la rama seleccionada
  • git fetch <remote> captura la historia del repositorio remoto
